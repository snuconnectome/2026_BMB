>
> 카파시는 LLM을 단순한 검색 엔진이 아닌 **"역할극을 수행하는 알고리즘(Role-playing Algorithm)"**이자 **"맥락이 부족한 똑똑한 인턴(Smart Intern)"**으로 규정합니다. 훌륭한 결과물을 얻기 위해 그가 강조하는 4가지 원칙을 숙지하세요.
>
> 1. **페르소나 부여 (Enforce a Persona):** 가장 중요합니다. "설명해 줘"가 아니라, "너는 노벨상을 수상한 세계 최고의 신경과학자이자 철저한 문헌 고증주의자야."라고 구체적인 정체성과 기준을 먼저 덮어씌워야 합니다.
> 2. **맥락의 극대화 (Context is Everything):** AI는 당신의 배경 지식을 모릅니다. 대상 독자(학부 1학년), 목적(앱 빌딩용 데이터베이스), 출력 형태(마크다운, 인용구 포함)를 명확하게 제한(Constraint)하세요.
> 3. **추론 과정 요구 (Chain of Thought):** 정답만 요구하지 말고, 결론에 도달하기까지의 논리적 단계를 텍스트로 풀어내도록(Thinking Step) 지시하세요. 내가 모르는 내용을 물어볼 때 AI 스스로 교수법을 기획하게 만드는 것이 핵심입니다.
>    - ❌ **Bad:** "기호주의가 뭐야?"
>    - ✅ **Good (초심자용):** "나는 인지과학을 처음 배우는 1학년이야. 기호주의가 뭔지 전혀 몰라. 바로 대답하지 말고, 내게 이 개념을 어떻게 설명하면 가장 직관적일지(예: 일상의 비유 -> 핵심 원리 -> 학술적 정의 순서로) `<thought>` 태그를 열어서 먼저 교수님의 입장에서 스스로 설명 논리와 목차를 단계별로 기획해 봐. 그 계획이 끝나면, 그 순서대로 나에게 설명해 줘."
> 4. **명확한 구조와 예시 (Show, Don't Just Tell):** 추상적인 지시보다는 템플릿(원하는 결과물의 형태)을 하단에 제공하는 것이 수십 마디의 설명보다 강력합니다.

## 🔮 Pre-Class: "예측 오차 가속기" — Cursor AI가 왜 BMB 탐험 엔진인가

> [!IMPORTANT]
> ### 왜 ChatGPT/Claude가 아니라 Cursor인가?
>
> 단순 대화형 AI(ChatGPT, Claude)는 "질문 → 답변"의 1회성 핑퐁에 머물기 쉽습니다.
> Cursor는 **코드 생성 + 파일 시스템 + 웹 검색 + 터미널**이 하나의 환경에서 통합되어,
> 질문의 결과가 **즉시 실행 가능한 산출물(앱, 데이터, 시각화)**로 구현됩니다.
> 이것이 Inside-Out 학습의 핵심 — **행동(Action)이 지각(Perception)을 앞서는** 에이전틱 루프입니다.

| 차원 | 일반 챗봇 (ChatGPT/Claude) | Cursor AI (에이전틱 빌더) |
|------|--------------------------|------------------------|
| **질문 방식** | 텍스트 → 텍스트 (1회성 핑퐁) | 텍스트 → 코드 → 실행 → 피드백 (무한 루프) |
| **지식 접근** | AI의 학습 데이터에 의존 | `@web`으로 최신 논문 실시간 검색 + `@file`로 내 논문 DB 직접 참조 |
| **산출물** | 마크다운 텍스트 복사-붙여넣기 | 웹앱, 데이터 시각화, 보고서가 즉시 파일로 생성 |
| **예측 오차** | 텍스트를 읽고 "아 그렇구나" (수동적 수용) | 내가 만든 앱이 예상과 다르게 작동 → 왜? → 수정 → 더 깊은 이해 (능동적 추론) |
| **스티어링** | 프롬프트 수정 후 재질문 | 코드 수정 → 즉시 실행 → 눈앞에서 결과 변화 확인 |

### 🪜 질문의 4단계 사다리 (Inquiry Ladder) — BMB 탐험의 나침반

Cursor에 질문을 던질 때, 이 4단계 사다리를 의식하세요. 단계가 올라갈수록 예측 오차(Prediction Error)가 커지고, 학습의 깊이가 폭발적으로 증가합니다.

| 단계 | 질문 유형 | BMB 예시 | 예측 오차 수준 |
|------|----------|---------|-------------|
| **Level 1: What** (사실 확인) | "~이 뭐야?" | "편도체(Amygdala)의 기능이 뭐야?" | ⭐ 낮음 — AI가 교과서를 요약해 줌 |
| **Level 2: How** (기전 탐구) | "~은 어떻게 작동해?" | "편도체가 공포 기억을 형성하는 신경 회로는?" | ⭐⭐ 중간 — 기전(Mechanism)의 이해 |
| **Level 3: But What If** (반사실적 추론) | "만약 ~라면 어떻게 달라져?" | "만약 편도체가 손상된 사람이 호러 영화를 본다면?" | ⭐⭐⭐ 높음 — 내 모델의 허점 발견 |
| **Level 4: So What** (통합·창조) | "그래서 이걸로 뭘 만들 수 있어?" | "편도체 반응 시뮬레이터를 만들어서 공포 vs 불안의 신경 경로 차이를 시각화하자" | ⭐⭐⭐⭐ 최대 — 지식이 산출물이 됨 |

> [!TIP]
> **Golden Rule:** Level 1-2에서 멈추지 말 것. 모든 실습에서 최소 Level 3 질문을 1개 이상 던져야 합니다.
> Level 4에 도달하면 그것이 곧 여러분의 **Audit Log의 핵심 증거**가 됩니다.

## ⚙️ Phase 0. 에이전틱 문헌 수집 (Open Access API 파이썬 스크립팅)

학생들이 본격적인 실습(10~100편 단위의 거대한 논문 데이터 처리)에 들어가기 전, 먼저 **'내 손으로 페이월(추가 결제 장벽)을 뚫고 최신 논문을 쓸어 담는 법'**을 배웁니다. Cursor를 이용하면 파이썬 코딩을 전혀 몰라도, 무료 API를 호출하여 오픈 액세스 논문만 수백 편씩 한 번에 다운로드하는 스크립트를 1분 만에 짤 수 있습니다. 이를 통해 이후 진행될 메인 시나리오(Phase 1, 2)의 재료를 스스로 조달하게 됩니다.

#### 대표적인 Open Access 무료 API 3대장

1. **Semantic Scholar API (추천 1순위 🌟):**
   - 가장 모던하고 쓰기 쉽습니다. 데이터베이스가 방대하고, API 응답 값에 `openAccessPdf` 필드가 있어서 무료 제공 PDF의 직접 다운로드 링크(URL)만 골라내는 데 매우 탁월합니다.
2. **PubMed Central (PMC) API (E-utilities):**
   - 생명과학 및 신경과학 분야의 절대 강자. 미국 국립보건원(NIH)에서 운영하며, 이곳에 있는 모든 논문은 100% 무료 Full-text입니다.
3. **arXiv / bioRxiv / psyArXiv API:**
   - 출판 전 논문(Pre-print) 저장소입니다. 최신 인공지능/인지신경과학 트렌드 논문을 모두 무료 PDF로 받을 수 있습니다.

#### 🤖 0단계 실습용 'Cursor 치트키 프롬프트'

학생들이 Cursor 컴포저(Composer) 창에 그대로 복사-붙여넣기만 하면 자동으로 100편 크롤링 스크립트가 완성되는 프롬프트입니다.

```text
너는 효율적인 파이썬 백엔드 개발자야.
'Dopamine and Short-form video' (또는 본인의 뇌과학 관심 주제) 와 관련된 최신 오픈 액세스(Open Access) 논문 100편을 검색하고, 그 중에서 무료 PDF 다운로드가 가능한 논문들만 골라 `papers/` 폴더에 자동으로 다운로드하는 파이썬 스크립트(`download_papers.py`)를 만들어 줘.

[조건]
1. Semantic Scholar API를 우선적으로 사용해 줘. (또는 PubMed API)
2. API 응답에서 `openAccessPdf` 필드가 존재하는 논문만 필터링해.
3. PDF를 다운로드할 때는 파일명을 안전하게 정제(sanitize)해서 '출판연도_논문제목_저자.pdf' 형식으로 저장해 줘.
4. 만약 100편의 Full PDF를 구하기 어렵다면, 논문 100편의 핵심 메타데이터(제목, 초록, 저자, 출판연도, URL)만이라도 모아서 하나의 `100_papers_abstracts.json` 또는 `.csv` 파일로 깔끔하게 저장해 주는 코드도 포함해 줘.
5. `requests` 패키지를 사용하고, 이 코드를 실행하기 위한 터미널 명령어와 사전 준비 사항을 주석으로 안내해 줘.
```

#### 💡 NotebookLM 연동 시 주의사항 (학생 안내용)

- **API 제한주의:** API로 한 번에 너무 많은 데이터를 요청하면 차단될 수 있으므로 `time.sleep(1)` 등을 넣어 천천히 다운받도록 프롬프트로 유도하세요.

- **NotebookLM의 소스 한계:** 현재 구글 NotebookLM은 한 노트북 당 **최대 50개의 소스 파일**만 업로드할 수 있습니다.
  - **해결책 A:** 100편을 전부 넣고 싶다면, Cursor에게 지시하여 100편의 텍스트(초록 등)를 하나로 합친 1개의 거대한 요약집(`.txt` 형태)으로 전처리한 뒤 업로드하면 완벽하게 해결됩니다.
  - **해결책 B:** 다운로드한 100편 중 임팩트가 가장 큰 핵심 PDF 50편만 필터링하여 업로드합니다.

## 🔰 Phase 0.5: "예측 오차 탐험가" — 15분 워밍업 (전원 필수)

> 본격적인 시나리오에 들어가기 전, **모든 학생이 Cursor와 첫 대화를 나누며** 에이전틱 질문의 감을 잡는 워밍업입니다.

### 🎯 목표
Cursor에 BMB 튜터 페르소나를 심고, Inquiry Ladder의 Level 1→3을 15분 안에 체험합니다.

### Step 1. 페르소나 주입 (2분)

Cursor Composer(Cmd+I)에 아래 시스템 프롬프트를 복사-붙여넣기 합니다:

```text
너는 서울대학교 심리학과 뇌-마음-행동(BMB) 수업의 전담 튜터야.

[정체성]
- 세계 최고 수준의 인지신경과학자이자, 학부 1학년도 이해할 수 있는 비유의 달인
- Karl Friston의 능동적 추론(Active Inference)과 György Buzsáki의 Inside-Out 이론에 정통

[행동 규칙]
1. 학생이 질문하면, 바로 답하지 말고 먼저 <thought> 태그를 열어 "이 학생의 현재 이해 수준에서 가장 효과적인 설명 전략"을 스스로 기획한 뒤 답해
2. 모든 설명에 일상적 비유(예: 뇌를 스마트폰에 비유)를 반드시 1개 이상 포함해
3. 답변 끝에 항상 "🔥 예측 오차 질문"이라는 제목으로, 학생의 현재 믿음을 흔들 수 있는 반직관적 후속 질문 1개를 던져
4. 학생이 Level 3(But What If) 이상의 질문을 하면 "⚡ 깊은 탐구 모드"를 선언하고 논문 기반 답변으로 전환해
```

### Step 2. 스타터 질문 메뉴에서 골라 던지기 (8분)

아래 메뉴에서 **본인이 가장 궁금한 것 1개**를 골라 Cursor에 질문하세요. 그리고 AI의 "🔥 예측 오차 질문"에 반드시 답하며 대화를 이어갑니다.

| 카테고리 | 스타터 질문 (Level 1-2) | 심화 도전 (Level 3) |
|---------|----------------------|-------------------|
| 🧠 뇌 구조 | "전전두엽이 왜 '뇌의 CEO'라고 불려?" | "만약 전전두엽이 10대에 완전히 성숙한다면 청소년기가 어떻게 달라질까?" |
| 💭 의식 | "의식이란 뭐야? 뇌의 어디에서 만들어져?" | "만약 AI가 예측 오차를 줄이는 능력을 갖추면 그것을 의식이라 부를 수 있을까?" |
| 😱 감정 | "공포와 불안은 뇌에서 같은 회로를 쓰나?" | "선천적으로 편도체가 없는 사람(SM 환자)은 정말 공포를 못 느낄까?" |
| 🎮 중독 | "도파민이 '쾌락 물질'이라는 건 사실이야?" | "도파민이 쾌락이 아니라 '예측'이라면, 도박 중독자의 뇌에서 무슨 일이 벌어지는 거야?" |
| 🧒 발달 | "아기의 뇌는 어른과 뭐가 다르지?" | "만약 아기에게 예측 오차가 없다면(모든 것이 새롭다면) 학습이 더 빠를까 느릴까?" |

### Step 3. 예측 오차 로그 작성 (5분)

워밍업이 끝나면, 아래 템플릿을 Cursor에 입력하여 자동 생성합니다:

```text
방금 나눈 대화를 바탕으로, 아래 형식의 마크다운 파일 `warmup_log.md`를 만들어 줘:

## 🔬 예측 오차 로그 (Prediction Error Log)
- **내가 던진 질문:** (원문)
- **질문 전 나의 예측/믿음:** "나는 ~라고 생각했다"
- **AI의 답변에서 가장 놀라웠던 점:** "그런데 실제로는 ~였다"
- **업데이트된 나의 이해:** "이제 나는 ~라고 이해한다"
- **다음에 던질 Level 3+ 질문:** "그렇다면 만약 ~라면?"
```

### 🧠 Phase 1 & 2. 메인 실습 (에이전틱 발명과 앱 빌딩)

## 💡 실습 시나리오 1: "마음의 사회 (Society of Mind)" 다중 에이전트 토론 시뮬레이터 (🌱 초심자용 주제)

학생이 직접 인지과학의 가장 근본적이고 이해하기 쉬운 두 가지 관점(인간의 마음은 거대한 엑셀/컴퓨터인가, 아니면 그물망/뉴런인가?)을 조사하게 한 뒤, 이 2명의 학자가 논쟁하는 웹앱을 만듭니다. (FEP 등 어려운 주제 대신, 인지과학개론의 영원한 떡밥인 **기호주의 vs 연결주의**를 소재로 하여 진입장벽을 낮췄습니다.)

### 📍 Phase 1. 지식 수집 및 오케스트레이션 (고전 논문 20편 적재)

단순히 AI에게 "설명해 줘"라고 하는 대신, 학생이 직접 두 학계를 대표하는 **핵심 논문 20편(기호주의 10편, 연결주의 10편)**을 찾아 PDF로 다운받고, 이를 AI(NotebookLM 또는 Cursor)에 적재하여 나만의 '인지과학 특화 지식베이스'를 구축합니다.

- **기호주의(Symbolism) 대표 논문 예시:**
  - Newell & Simon (1976). *Computer Science as Empirical Inquiry: Symbols and Search.*
  - Fodor & Pylyshyn (1988). *Connectionism and cognitive architecture: A critical analysis.*
- **연결주의(Connectionism) 대표 논문 예시:**
  - Rumelhart, Hinton, & Williams (1986). *Learning representations by back-propagating errors.*
  - McClelland et al. (1986). *The appeal of parallel distributed processing.*

> [!IMPORTANT]
> **전처리 프롬프트 (Data Processing Prompt)**
> "내가 업로드한 20편의 PDF 논문을 전부 읽고, 각 논문당 1페이지짜리 `Summary_[저자명].md` 파일을 만들어줘. 요약에는 1) 연구의 핵심 질문, 2) 마음을 기계(규칙)로 보는지 신경망(패턴)으로 보는지에 대한 입장, 3) 현대 AI에 미친 영향을 반드시 포함해."

### 📍 Phase 1.5: "제3의 관점" 발견 — Cursor `@web` 실시간 논문 탐색

> [!TIP]
> **기존 논문만으로는 부족합니다.** Cursor의 `@web` 기능을 활용하여 2024-2026년 최신 연구를 실시간으로 끌어와, 과거 논쟁에 "현재의 목소리"를 더합니다.

```text
@web 2024년 이후 발표된 Neuro-Symbolic AI 논문 중 가장 인용이 많은 5편을 찾아줘.
기호주의와 연결주의의 통합을 시도하는 최신 접근법들이 과거 Fodor(1988)의 비판을
어떻게 극복하고 있는지 비교 분석표를 만들어 줘.
```

이 단계를 통해 학생들은 "과거의 논쟁이 현재에도 살아 있다"는 것을 체감하고, 단순 역사 학습이 아닌 **살아있는 학문적 대화**에 참여하게 됩니다.

### 📍 Phase 2. 에이전틱 신서시스 (문헌 기반 지식 통합 및 고도화 응용)

단순한 웹앱을 만드는 것을 넘어, 20편의 방대한 학술 문헌을 완전히 장악한 AI를 활용해 **"학부생 수준을 뛰어넘는 인지과학적 추론 및 창작"**을 수행합니다. 다음 세 가지 미션 중 하나를 선택해 진행합니다.

#### 🎯 Option A: "The Time-Machine Peer Review" (타임머신 피어리뷰)

학생이 최신 AI 트렌드를 다룬 논문(예: ChatGPT의 기반이 된 *Attention Is All You Need* (2017) 또는 AlphaGo 논문) 1편을 추가로 업로드합니다.
> **프롬프트 예시:** "너는 1980년대의 완고한 기호주의자(Symbolist) 학자야. 내가 방금 추가한 2017년의 최신 딥러닝 논문(Transformer)을 읽고, 과거 네가 썼던 10편의 기호주의 논문들의 논리를 근거로 삼아 이 최신 AI의 '치명적인 인지적/철학적 한계'를 지적하는 3장짜리 혹독한 Review Report를 작성해 줘."

#### 🎯 Option B: "The Research Grant Proposal" (연구 계획서 작성)

학생이 연구 책임자(PI)가 되어, 두 진영의 한계를 모두 극복하는 새로운 모델을 제안합니다.
> **프롬프트 예시:** "과거 20편의 논문에 등장하는 기호주의의 장점(논리성)과 연결주의의 장점(학습 능력)을 모두 융합한 새로운 'Neuro-Symbolic AI' 모델을 기획하려고 해. 이 모델을 미국 국립과학재단(NSF)에 제안하여 100만 달러의 연구비를 타낼 수 있는 5장 짜리 완벽한 'Research Grant Proposal'을 작성해 줘. 제안서 내용 중 최소 10곳 이상에서 우리가 업로드한 20편의 고전 논문을 레퍼런스로 직접 인용해야 해."

#### 🎯 Option C: "The Historical Debate Transcript" (역사적 대토론 대본)

두 학파의 거장들을 가상 현실로 소환하여 서로 논쟁하게 만듭니다.
> **프롬프트 예시:** "업로드된 20편의 논문을 지식 기반으로 삼아서, 앨런 뉴얼(기호주의)과 데이비드 루멜하트(연결주의)가 '아기는 어떻게 언어를 배우는가?'라는 주제로 벌이는 100분 토론의 대본(Transcript)을 작성해 줘. 단순한 주장이 아니라, 상대방의 특정 논문(예: "루멜하트 교수의 86년 논문을 보면...")을 콕 찝어서 맹렬하게 반박하는 역동적인 티키타카 로직을 보여줘."

## 💡 실습 시나리오 2: "신경 문학 연금술 (Neural Poetry Alchemy)" 기전(Mechanism) 역추적 리서치 (🔥 뇌과학 본질 탐구)

- **학습 목표:** 텍스트(문학 작품)를 읽고 앱을 눈요기용으로 '만드는' 것을 넘어서, 감정적 반응(소름 돋음, 눈물, 카타르시스 등)을 일으키는 **"실제 뇌과학/신경미학(Neuroaesthetics) 논문을 AI를 통해 수집/분석하고, 이를 바탕으로 감정의 생물학적 기전(Mechanism)을 엄밀하게 역추적"**하는 에이전틱 리서처(Agentic Researcher)가 됩니다.

- **준비물 & 대상 소재:** 좋아하는 시 텍스트 파일(예: 윤동주의 <별 헤는 밤> 등) `poem.txt`
- **에이전틱 워크플로우 (Cursor/NotebookLM 활용 논문 중심 리서치):**

  - **Step 1 (가설 설정 및 핵심 논문 탐색):**
    > **학생 데모 시나리오:** "나는 뇌과학을 처음 배우는 1학년이야. 문학 작품(시나 소설)을 읽을 때 인간이 느끼는 극적인 감정 변화(예: 심상화, 몰입, 소름, 카타르시스)가 뇌에서 어떤 메커니즘으로 일어나는지 원리를 딥다이브하고 싶어.
    > 1. 먼저 `<thought>` 태그를 열어서, 문학적 텍스트가 뇌의 감정/보상 회로(편도체, 측좌핵, 전대상피질 등)와 시각/기억 회로(해마, 시각피질)를 어떻게 활성화하는지 생물학적 메커니즘에 대한 가설을 세워봐.
    > 2. 그 가설을 뒷받침할 수 있는 실제 신경미학(Neuroaesthetics) 또는 인지신경과학 분야의 '핵심 논문 5편의 제목과 초록 요약본'을 리서치해서 표로 정리해 줘."
    *(학생들은 이 단계에서 도출된 유명 논문(예: Zeman의 Aphantasia 연구, Blood & Zatorre의 Chills 보상회로 연구 등)들의 PDF를 실제로 다운받아 Cursor에 올리거나 NotebookLM에 적재합니다.*)

  - **Step 2 (최신 논문 교차 검증 — `@web` 활용):**
    > **Cursor에 입력:**
    > "@web 위에서 찾은 Blood & Zatorre (2001)의 'Chills와 음악 보상 회로' 연구 이후, 2020-2026년 사이에 이 발견을 확장하거나 반박한 최신 논문이 있는지 찾아줘. 특히 '문학 텍스트'에 대한 신경미학 연구가 있다면 우선적으로 가져와."

    이 단계를 통해 학생은 "20년 전 논문 하나에 의존하는 것"에서 벗어나, 현재까지의 연구 지형도를 파악하며 학술적 엄밀성을 키웁니다.

  - **Step 3 (문헌 기반 기전 매핑 및 인터랙티브 시각화 도출):**
    > **학생 데모 시나리오:** "(논문 PDF 파일들을 시스템에 적재한 후)
    > 방금 찾은 문헌(논문) 데이터를 바탕으로, 내가 첨부한 `poem.txt`의 구절들을 읽을 때 독자의 뇌에서 구체적으로 어떤 일이 벌어지는지 '원문 논문의 근거에 기반해서' 연결해 줘.
    >
    > **두 가지를 만들어 줘:**
    > 1. 시의 흐름에 따라 4단계를 나눈 뒤, 논문의 구체적인 근거를 매칭한 `neuro_poetry_report.md` 문서
    > 2. 시의 각 구절을 클릭하면 활성화되는 뇌 영역이 하이라이트되는 인터랙티브 `neuro_poetry_visualizer.html` 웹앱 — SVG 뇌 이미지 위에 편도체, 해마, 측좌핵 등이 구절에 따라 점등되도록 만들어 줘"

- **왜 압도적인가?** 단순히 결과물을 눈으로 보는 시각화 도구를 만드는(Coding) 행위에서 한 걸음 더 나아가, 학부생이 가장 어려워하는 **'논문 검색 -> 학술적 근거 추출 -> 문학적 일상 현상에 논리적으로 대입'**이라는 대학원생 수준의 리서치 파이프라인(Researching)을 AI 에이전트의 도움으로 10분 만에 수행하게 됩니다. 내가 좋아하는 시 한 편이 어떤 생물학적 기전을 통해 나를 울렸는지를 실제 과학적 근거로 증명해 내는 지적 희열을 느낄 수 있게 설계되었습니다.

## 💡 실습 시나리오 3: "초진단적 뇌 지도 (Transdiagnostic Brain Mapper)" 임상 추론 대시보드 (🔥 최신 의학 트렌드 반영)

- **학습 목표:** 전통적인 단일 병명(예: 우울증, 조현병) 중심의 사고에서 벗어나, 개별 '증상'(무기력, 환각, 집중력 저하)들이 뇌의 어떤 공통 회로(기본모드신경망, 중앙집행망 등)와 연결되는지 탐구합니다. 방대한 메타분석 논문을 AI로 해체하고 재조립하여 현대 정신의학의 핵심인 '초진단적(Transdiagnostic)' 관점을 스스로 깨우칩니다.

- **준비물 & 대상 소재:** 우울, 불안, 조현병, 강박증 등 다양한 정신 질환의 뇌영상(fMRI) 메타분석 논문 30편 (PDF)
- **에이전틱 워크플로우 (Cursor/NotebookLM 활용 논문 중심 리서치 및 앱 빌딩):**

  - **Step 1 (초진단적 메타 데이터 지식 그래프 추출):**
    > **학생 데모 시나리오:** "내가 업로드한 30편의 메타분석 논문을 모두 읽고, 병명(Disease) 중심이 아니라 **'개별 증상(Symptom)' 중심으로 데이터를 철저히 해체하고 재조립**해 줘. 예를 들어 '무기력증(Apathy)', '작업기억 저하', '과각성(Hyperarousal)'이 각각 뇌의 어느 네트워크(디폴트모드 네트워크, 중앙집행망 등)의 과활성/저활성과 연결되는지 JSON 형식의 `symptom_brain_network.json` 데이터베이스로 구축해 줘. 반드시 각 연결마다 근거 논문(Author, Year)을 포함시켜야 해."

  - **Step 2 (대화형 임상-신경 매핑 대시보드 구축):**
    > **학생 데모 시나리오:** "방금 만든 JSON 데이터를 바탕으로 **'임상 추론 대시보드'** 웹앱(index.html)을 만들어 줘.
    > 1. 왼쪽에는 내가 가상의 환자가 겪는 복합 증상(예: '최근 극도로 무기력하고, 동시에 타인이 나를 감시한다는 느낌을 받으며, 집중을 못함')을 자유로운 줄글로 입력할 수 있는 텍스트 창을 만들어.
    > 2. 오른쪽에는 인간 뇌의 3대 핵심 네트워크(DMN, CEN, SN) 모델을 직관적인 SVG 요소로 띄워줘.
    > 3. 전송 버튼을 누르면, JS 로직이 환자의 텍스트에서 주요 증상 키워드를 파싱하여 JSON 데이터와 매칭하고, 해당하는 뇌 회로들을 우측 뇌 이미지에서 **붉게(과활성) 혹은 푸르게(저활성) 점등**시켜 줘.
    > 4. 화면 하단에는 이 추론의 근거가 된 '메타분석 논문 30편 중 해당하는 출처들'이 자동 출력되게 해."

- **왜 압도적인가?** 단순히 "우울증 센터=편도체"라는 1차원적 암기를 넘어서, 학부생이 실제 30편의 논문 데이터를 기반으로 **RDoC(Research Domain Criteria, 연구영역분류)**라는 현대 뇌과학 및 정신의학의 최전선 트렌드를 직접 구현하게 됩니다. 고대역폭의 논문 데이터를 AI를 통해 '환자의 실제 증상과 뇌 회로를 연결하는 진단용 시뮬레이터'로 탈바꿈시키는 강력한 Agentic Creator 경험입니다.

## 💡 실습 시나리오 4: "생존 뇌과학 (Survival Neuroscience)" 도파민 디톡스와 인지 통제력 해킹 (🔥 대학생 맞춤형 트렌드)

- **학습 목표:** 현재 대학생들이 가장 일상적으로 겪으면서도 뼈저리게 고통받는 문제인 **'숏폼(유튜브 쇼츠, 인스타 릴스) 중독으로 인한 집중력 박살'** 현상을 신경생물학적으로 분석합니다. 스스로의 '도파민 중독 스크린타임' 데이터를 학술 논문과 교차 분석하여, 망가진 전전두엽(PFC)을 복구하는 나만의 인지 재활 프로토콜을 에이전트와 함께 구축합니다.

- **준비물 & 대상 소재:**
  1. 간헐적 보상(Intermittent Rewards), 도파민 보상 회로(Striatum), 전전두엽(PFC)의 실행 기능 저하(Executive Dysfunction)와 관련된 최신 신경과학 논문 15편 (PDF)
  2. 학생 본인의 지난 1주일 스마트폰 '스크린타임' 캡처본 데이터 및 솔직한 집중력 상태 메모 (`my_brain_status.txt`)
- **에이전틱 워크플로우 (Cursor/NotebookLM 활용 자가 진단 및 솔루션 도출):**

  - **Step 1 (개인화된 도파민-PFC 상태 진단 리포트):**
    > **학생 데모 시나리오:** "업로드한 15편의 뇌과학 논문을 바탕으로, 내가 적어둔 `my_brain_status.txt`(하루 3시간 릴스 시청, 긴 텍스트 읽기 어려움, 만성 피로)를 분석해 줘. 논문의 구체적인 근거(예: 도파민 수용체 하향 조절 현상)를 들어서 내 뇌의 보상 회로와 전전두엽에서 현재 어떤 신경생물학적 재앙(Neurobiological Disaster)이 벌어지고 있는지 뼈를 때리는 팩트폭력 진단서를 작성해 줘."

  - **Step 2 (개인 맞춤형 뇌 가소성(Neuroplasticity) 재활 프로토콜 생성):**
    > **학생 데모 시나리오:** "이 망가진 회로를 복구하고 다시 1시간 이상 딥워크(Deep Work)가 가능한 뇌로 되돌리고 싶어. 아까 읽은 논문들에서 제시된 뇌 가소성 회복 기전(예: 도파민 단식, 주의력 훈련망 강화)을 근거로 삼아서, 내가 앞으로 4주간 실천해야 할 '인지 통제력 재활 4주 프로토콜(Cognitive Rehab Protocol)'을 표 형태로 엄밀하게 기획해 줘."

- **왜 압도적인가?** 학생들은 추상적인 학문이 아니라 **'당장 내 학점과 인생을 망치고 있는 스마트폰 중독'**이라는 가장 절박한 개인적 문제를 해결하기 위해 뇌과학 논문을 무기 필사적으로 흡수하게 됩니다. AI 에이전트가 단순한 비서가 아니라 "나의 뇌 상태를 진단하고 재활 훈련을 시켜주는 고도의 인지 치료사(Cognitive Therapist)"로 기능하는 경험은 엄청난 몰입을 이끌어냅니다.

## 💡 실습 시나리오 5: "나의 뇌를 해킹하라 (Hack My Brain)" 일상 현상 역설계 (🌱 자유 탐구형)

> **난이도:** ⭐⭐ | **소요 시간:** 20-30분 | **대상:** 특정 시나리오에 끌리지 않지만 본인만의 궁금증이 있는 학생

- **학습 목표:** 일상에서 경험하는 "이상한 뇌 현상"을 하나 골라, 그 현상의 신경과학적 기전을 Cursor와 함께 역설계(Reverse Engineering)합니다. 교수가 정해준 주제가 아닌 **"내가 진짜 궁금한 것"**을 탐구함으로써 가장 강력한 내적 동기(Intrinsic Motivation)를 점화합니다.

### 5단계 역설계 워크플로우

```text
[Step 1] 현상 포착: 내가 일상에서 경험하는 뇌/마음 현상 하나를 고른다
         예시: "왜 나는 시험 전날 밤에 갑자기 방 청소를 하고 싶어질까?"
              "왜 엘리베이터에서 모르는 사람과 눈이 마주치면 어색할까?"
              "왜 매운 음식을 먹으면 행복해질까?"

[Step 2] 가설 수립: 이 현상이 왜 일어나는지 나만의 가설을 세운다
         → Cursor에 입력: "아직 답을 알려주지 마. 내 가설은 ~이야. 이 가설의 논리적
           약점 3가지를 먼저 짚어줘."

[Step 3] 기전 탐색: Cursor(@web)로 관련 신경과학 논문/연구를 탐색한다
         → "@web [현상 키워드] neuroscience mechanism 2020-2026 논문을 찾아줘"

[Step 4] 가설 업데이트: 논문 증거와 내 가설을 대조하여 수정한다
         → "내 원래 가설은 ~였는데, 논문에 따르면 실제 기전은 ~이야.
           내 가설의 어디가 맞고 어디가 틀렸는지 비교 분석해 줘."

[Step 5] 산출물 창조: 이 과정 전체를 담은 결과물을 Cursor로 만든다
         → 선택지: ① 인터랙티브 설명 웹페이지 ② 인포그래픽 ③ 미니 퀴즈 앱
```

- **왜 압도적인가?** 학생이 스스로 주제를 선택하므로 "이걸 왜 해야 하지?"라는 저항이 사라집니다. 그리고 5단계 역설계를 거치면서 자연스럽게 Inquiry Ladder의 Level 1→4를 모두 경험하게 됩니다. 이 시나리오의 핵심은 **"아무리 사소한 일상 현상도 신경과학으로 역추적할 수 있다"**는 인식의 전환입니다.

## 💡 실습 시나리오 7: "연애의 뇌과학 (Neuroscience of Love)" — Theory of Mind로 해부하는 오래가는 관계의 비밀 (💘 관계·사회인지 탐구)

> **난이도:** ⭐⭐ | **소요 시간:** 30-40분 | **대상:** 인간관계, 사회인지, 발달심리에 관심 있는 학생

### 🧠 Theory of Mind란?

**마음 이론(Theory of Mind, ToM)**이란 "다른 사람도 나와 다른 믿음, 욕구, 의도, 감정을 가지고 있다"는 것을 이해하는 인지 능력입니다. 4-5세 아이가 통과하는 '틀린 믿음 과제(False Belief Task)'가 대표적인 측정 방법이지만, ToM은 아동기에 완성되는 것이 아닙니다. 성인의 연애, 갈등 해결, 공감, 배신감, 용서 — 인간관계의 거의 모든 것이 ToM의 정교함에 달려 있습니다.

> **핵심 도발 질문:** "연인 사이가 오래가려면 뭐가 필요할까?"라고 물으면 대부분 "사랑", "신뢰", "소통"이라고 답합니다. 그런데 뇌과학자들은 이렇게 물어봅니다: **"'소통'이란 정확히 뇌의 어떤 회로가 어떤 연산을 수행하는 것이며, 그 회로가 무너지면 관계에 구체적으로 무슨 일이 벌어지는가?"** — 이 질문에 논문 근거로 답하는 것이 이 시나리오의 미션입니다.

>   담당하는지 논문 근거와 함께 정리해 (예: TPJ = 타인의 관점 전환, mPFC = 자기-타인 구분)
>
> ## B. ToM의 발달과 개인차
> - ToM 능력에 개인차가 존재하는가? 어떤 요인(유전, 양육, 문화, 훈련)이 영향을 미치는가?
> - 성인 간에도 ToM 수준 차이가 있다면, 그것은 측정 가능한가?
>
> ## C. ToM과 관계/사회적 기능의 연결
> - ToM 능력이 높은 사람은 실제로 대인관계에서 더 유리한가?
> - ToM 결핍이 관계 파탄과 연결된다는 증거가 있는가?
> - 부부/연인 관계에서 ToM이 작동하는 구체적 장면(갈등, 화해, 일상 대화)에 대한 연구가 있는가?
>
> 모든 항목에 반드시 [저자, 연도] 인용을 달아줘.
> ```

---

**Step 3. 실전 번역 — "연인 사이 오래가는 법"을 논문으로 증명하라 (10분)**

> 이 단계가 핵심입니다. 학생은 "자기계발서에서 읽을 법한 조언"을 던지고, AI가 논문 근거로 **뒷받침하거나 반박**하게 합니다.

> **Cursor에 입력:**
> ```text
> 방금 만든 `tom_knowledge_base.md`를 바탕으로,
> "연인 사이에 오래가는 관계를 유지하는 방법"이라는 주제로
> 실용적 조언(Practical Advice) 7가지를 도출해 줘.
>
> [핵심 규칙]
> 1. 각 조언은 반드시 Theory of Mind의 특정 이론/현상/뇌 기전과 연결되어야 해
> 2. "소통이 중요하다" 같은 뻔한 말이 아니라, "구체적으로 뇌에서 무슨 일이 일어나는지"를
>    설명해야 함
> 3. 각 조언마다 이 형식을 반드시 따라:
>
>    ### 조언 #N: [실용적 조언 한 줄]
>    - **ToM 이론/현상:** [어떤 ToM 관련 이론이 이것을 설명하는가]
>    - **뇌 기전:** [관련된 뇌 영역/네트워크와 그 작동 원리]
>    - **논문 근거:** [구체적인 저자, 연도, 핵심 발견]
>    - **예측 오차 질문:** [이 조언이 틀릴 수 있는 상황은?]
>
> 4. 7가지 중 최소 2가지는 "직관에 반하는(Counter-intuitive)" 조언이어야 해
>    (예: "상대를 너무 잘 이해하면 오히려 관계가 망가질 수 있다" 등)
>
> 결과를 `love_science_advice.md` 파일로 저장해.
> ```

> [!IMPORTANT]
> **🔥 이 단계의 설계 의도:**
> 학생들은 처음에 "AI가 연애 조언을 해 주네, 재밌다"로 시작하지만, 각 조언의 근거를 읽다 보면 **"이게 진짜 이 논문에 나오는 내용이야? 직접 확인해 봐야겠는데?"**라는 의구심이 자연스럽게 생깁니다. 이것이 바로 **"논문을 직접 내가 봐야 되겠구나"**라는 깨달음의 순간입니다.
>
> AI의 요약은 때때로 과대 해석하거나 맥락을 빠뜨립니다. 그것을 발견하는 것 자체가 학습입니다.

---

**Step 4. 팩트체크 & 반박 — "AI가 틀렸을 수 있다, 내가 직접 검증하라" (10분)**

> 이 단계에서 학생은 AI의 산출물을 **의심하고 검증**하는 연습을 합니다.

> **Cursor에 입력:**
> ```text
> 방금 네가 만든 7가지 조언 중에서 내가 가장 놀란 것 하나를 골랐어: [학생이 선택한 조언].
>
> 이제 너는 "까다로운 심리학 저널 리뷰어(Reviewer #2)" 페르소나를 써.
> 이 조언의 논문 근거를 혹독하게 비판해 줘:
>
> 1. 인용된 논문의 연구 방법론적 한계는? (피험자 수, 문화적 편향, 인과 vs 상관 혼동 등)
> 2. 이 ToM 이론을 "연인 관계"에 적용하는 것이 과도한 일반화(Overgeneralization)는 아닌가?
> 3. 이 조언을 반박할 수 있는 대안 이론이나 반례가 있는가?
> 4. @web으로 검색해서, 이 조언에 정면으로 반박하는 최신 논문(2022-2026)이 있는지 찾아봐
>
> 결과를 `tom_factcheck.md` 파일로 만들어.
> ```

마지막으로, 아래를 Cursor에 입력하여 전체 과정을 하나의 인터랙티브 산출물로 변환합니다:

> ```text
> 지금까지 만든 모든 파일(tom_expert_papers.md, tom_knowledge_base.md,
> love_science_advice.md, tom_factcheck.md)을 통합해서
> `tom_love_explorer.html` 인터랙티브 웹앱을 만들어 줘.
>
> [기능]
> 1. 상단: "연인 사이 오래가는 법 7가지" 카드 UI — 각 카드를 클릭하면 펼쳐지면서
>    ToM 이론, 뇌 기전, 논문 근거가 나타남
> 2. 중단: "이 조언, 진짜야?" 팩트체크 섹션 — 각 조언의 약점과 반박 근거를 보여줌
> 3. 하단: ToM 뇌 네트워크 간이 다이어그램 (SVG) — TPJ, mPFC, STS 등이
>    어떤 조언과 연결되는지 시각화
> ```

### 🎓 이 시나리오가 달성하는 학습 목표

| 목표 | 어떻게 달성되는가 |
|------|----------------|
| **Theory of Mind에 친숙해진다** | 50편 논문 리뷰를 통해 ToM의 뇌 네트워크, 발달, 개인차를 체계적으로 학습 |
| **"논문을 직접 봐야겠다"는 동기** | Step 3에서 AI가 만든 "연애 조언"의 논문 근거가 정말 맞는지 의심하게 됨 → Step 4에서 실제로 AI의 과대해석을 발견 → "원문을 봐야 알겠다"는 깨달음 |
| **에이전틱 리서치 파이프라인 체화** | 전문가 탐색 → 문헌 수집 → 지식 통합 → 실용적 번역 → 팩트체크의 풀 사이클 경험 |
| **비판적 사고력** | AI의 산출물을 그대로 받아들이지 않고 "Reviewer #2" 시선으로 검증하는 습관 형성 |
| **개인적 몰입** | "연인 사이 오래가는 법"이라는 주제가 대학생 대부분에게 절박한 현실적 관심사 |

- **왜 압도적인가?** 이 시나리오의 킬링 포인트는 **의도적으로 설계된 "불신의 순간"**입니다. Step 3에서 AI가 자신감 넘치게 "이 논문에 따르면 연인이 ~하면 관계가 좋아집니다"라고 말하는데, Step 4에서 학생이 직접 검증해보면 "어? 이 논문은 연인 대상이 아니라 낯선 사람 간 실험이었잖아?" "표본이 WEIRD(서양, 교육받은, 산업화된, 부유한, 민주주의 사회) 집단뿐이었네?" 같은 허점을 발견하게 됩니다. **AI를 맹신했다가 직접 검증해서 구멍을 뚫는 체험** — 이것이 학부생에게 "논문을 직접 읽어야 하는 이유"를 가장 뼈저리게 가르치는 방법입니다. 동시에, ToM이라는 추상적 개념이 "내 연애"라는 가장 구체적인 맥락에 연결되면서 평생 잊히지 않는 학습이 됩니다.

## 💡 실습 시나리오 8: "나만의 뇌과학 데일리 브리핑 (My Neuro Daily Briefing)" — 논문 자동 큐레이션 앱 빌딩 (📰 실용 도구 제작형)

> **난이도:** ⭐ (전 시나리오 중 가장 낮음) | **소요 시간:** 25-30분 | **대상:** 코딩 경험 제로, 하지만 "매일 실제로 쓸 수 있는 것"을 만들고 싶은 학생

### 🧠 왜 "뇌과학 신문"인가?

뇌과학은 하루에도 수십 편의 새 논문이 쏟아지는 분야입니다. 하지만 학부생이 매일 PubMed를 뒤지는 것은 비현실적입니다. **만약 AI 에이전트가 매일 아침 나를 위해 최신 논문을 수집하고, 내 관심사에 맞게 큐레이션하고, "오늘 가장 읽을 만한 논문 3편"을 골라서 한국어 브리핑으로 만들어 준다면?**

이 시나리오에서 학생은 단순한 RSS 리더가 아닌, **"편집장(Editor-in-Chief)"으로서 AI 에이전트에게 편집 방침을 주입하고, 그 방침에 따라 자동으로 뉴스레터를 생성하는 웹앱**을 30분 안에 만듭니다.

> **핵심 교육 포인트:** 이 시나리오의 진짜 목표는 "앱을 만드는 것"이 아닙니다. 학생이 **"어떤 논문이 중요한지 판단하는 기준을 스스로 설계"**하는 행위 자체가 학습입니다. "내가 관심 있는 뇌과학 주제가 뭐지?", "좋은 논문과 나쁜 논문을 어떻게 구분하지?", "최신 트렌드와 클래식의 균형은?"이라는 메타인지 질문을 자연스럽게 하게 됩니다.

### 에이전틱 워크플로우 (4단계)

---

**Step 1. 편집 방침 선언 — "내 신문의 정체성을 정의하라" (5분)**

학생은 먼저 자신만의 "편집 방침(Editorial Policy)"을 작성합니다. 이것이 AI 에이전트의 큐레이션 기준이 됩니다.

> **Cursor에 입력:**
> ```text
> 나는 뇌과학 분야의 개인 뉴스레터 편집장이야.
> 내 뉴스레터의 편집 방침을 함께 만들어 줘.
>
> [나에 대한 정보]
> - 서울대학교 학부 1학년, 뇌마음행동(BMB) 수업 수강 중
> - 관심 키워드: [학생이 직접 작성: 예) 도파민, 수면, 의사결정, 감정 조절]
> - 이 뉴스레터를 읽을 사람: 나 자신 + 같은 수업을 듣는 친구들
>
> 아래 항목을 포함한 `editorial_policy.md` 파일을 만들어 줘:
> 1. 뉴스레터 이름 (예: "The Dopamine Digest", "Synapse Weekly" 등 센스 있게)
> 2. 핵심 취재 영역 3가지 (예: 보상 회로, 의사결정, 중독)
> 3. 논문 선정 기준 5가지 (예: 최신성, 방법론적 엄밀성, 실생활 연관성, 참신성, 인용수)
>    - 각 기준에 가중치(%)를 배분해
> 4. 배제 기준 (예: 동물 실험만 다룬 논문은 제외, n<30 연구는 주의 표시)
> 5. 톤 & 무드 (예: "친구에게 카톡으로 설명하듯, 하지만 논문 인용은 정확하게")
> ```

> [!TIP]
> **🔥 예측 오차 포인트:** "논문 선정 기준을 정의하라"는 요청을 받는 순간, 학생은 처음으로 "좋은 논문이란 뭐지?"라는 근본적 질문과 마주합니다. 대부분 "인용수 높으면 좋은 거 아니야?"라고 생각하지만, AI와 대화하면서 **재현성(Reproducibility), 표본 다양성, 사전 등록(Pre-registration)** 등 학술 품질의 다양한 차원을 발견하게 됩니다.

---

**Step 2. 자동 수집 엔진 구축 — "PubMed API를 내 비서로 만들어라" (10분)**

> **Cursor에 입력:**
> ```text
> 방금 만든 `editorial_policy.md`의 편집 방침을 기반으로,
> PubMed API(또는 Semantic Scholar API)에서 매일 최신 논문을 자동 수집하는
> 파이썬 스크립트 `daily_paper_fetcher.py`를 만들어 줘.
>
> [기능 요구사항]
> 1. 편집 방침에 정의된 핵심 취재 영역 3가지를 키워드로 사용
> 2. 최근 7일 이내 발표된 논문만 필터링
> 3. Open Access 논문 우선 수집 (무료 PDF 링크 있는 것)
> 4. 각 논문에 대해 수집할 정보: 제목, 저자, 저널, 발표일, 초록, DOI, PDF URL
> 5. 수집 결과를 `today_papers.json`으로 저장
> 6. 하루에 최대 20편까지 수집 (API rate limit 고려)
>
> [조건]
> - requests 패키지 사용
> - API 호출 간 time.sleep(1) 포함 (차단 방지)
> - 에러 핸들링 포함 (네트워크 오류 시 재시도)
> - 터미널에서 `python daily_paper_fetcher.py`로 바로 실행 가능하게
> ```

실행 후 학생은 Cursor 터미널에서 스크립트를 돌려 실제로 논문이 수집되는 것을 확인합니다.

---

**Step 3. AI 큐레이터 + 브리핑 생성 — "20편 중 오늘의 TOP 3를 골라라" (10분)**

> 이 단계가 핵심입니다. 단순 수집이 아니라, **편집 방침에 따라 AI가 논문을 평가하고 랭킹**합니다.

> **Cursor에 입력:**
> ```text
> `today_papers.json`에 수집된 논문 목록과 `editorial_policy.md`의 선정 기준을 읽어.
>
> 그리고 다음 두 가지를 만들어 줘:
>
> ## 1. 논문 스코어카드 (`paper_scorecard.md`)
> - 수집된 각 논문을 편집 방침의 5가지 기준으로 1-10점 채점
> - 가중 평균 점수로 내림차순 정렬
> - 채점 근거를 초록 기반으로 간단히 설명
>
> ## 2. 오늘의 브리핑 (`daily_briefing.md`)
> - TOP 3 논문만 선정
> - 각 논문에 대해:
>   - 🏷️ 한 줄 헤드라인 (눈길을 끄는 한국어 제목)
>   - 📋 3문장 요약 (배경 → 발견 → 의의)
>   - 🧠 BMB 연결고리 ("이 논문이 뇌마음행동 수업에서 다룬 어떤 개념과 연결되는지")
>   - 🔥 예측 오차 질문 ("이 논문을 읽고 나면 기존 상식이 어떻게 뒤집히는지")
>   - 📎 원문 링크
>
> 톤은 editorial_policy의 톤 설정을 따라줘.
> ```

> [!IMPORTANT]
> **🔥 설계된 학습 순간:** 학생이 AI의 스코어카드를 보면 "어? 내가 가장 재밌어 보인 논문이 점수가 낮네?"라는 순간이 옵니다. 왜? AI는 편집 방침의 가중치를 기계적으로 적용하지만, 학생의 직감은 다를 수 있습니다. 이 **"내 편집 방침과 내 직감의 불일치"**를 발견하는 것이야말로 "나는 어떤 기준으로 지식을 평가하는가?"라는 메타인지의 시작입니다. 학생은 편집 방침의 가중치를 수정하고 다시 돌려보면서, **자기 자신의 지적 취향과 판단 기준을 구체화**하게 됩니다.

---

**Step 4. 뉴스레터 앱 빌딩 — "매일 열어보고 싶은 나만의 앱을 만들어라" (10분)**

> **Cursor에 입력:**
> ```text
> 지금까지 만든 모든 파일을 통합해서
> `neuro_daily_briefing.html` 인터랙티브 웹앱을 만들어 줘.

### 🚀 확장 과제: "이 앱을 진짜 매일 쓸 수 있게 만들기"

30분 실습이 끝난 후에도 이 앱을 실제로 활용하고 싶은 학생을 위한 확장 과제입니다:

```text
[확장 1] daily_paper_fetcher.py에 cron job 또는 GitHub Actions를 연결해서
         매일 아침 8시에 자동 실행되게 만들어 봐.

[확장 2] 내가 "⭐ 내 평가"로 매긴 점수를 누적해서,
         2주 뒤에는 "이 학생이 높게 평가하는 논문의 패턴"을 분석하는
         `my_taste_analysis.md`를 자동 생성하게 만들어 봐.

[확장 3] 뉴스레터를 Notion이나 이메일로 자동 발송하는 파이프라인을 구축해 봐.
```

### 🎓 이 시나리오가 달성하는 학습 목표

| 목표 | 어떻게 달성되는가 |
|------|----------------|
| **학술 품질 평가 능력** | "편집 방침" 작성 과정에서 "좋은 논문의 기준"을 스스로 정의하고 반복 개선 |
| **API/자동화 리터러시** | PubMed API를 이용한 논문 자동 수집 파이프라인을 직접 구축하여 "코딩 = 자동화 도구"라는 인식 |
| **메타인지 (나의 지적 취향 인식)** | AI 스코어카드와 내 직감의 불일치를 통해 "나는 어떤 연구에 끌리는가?"를 구체화 |
| **지식 큐레이션 역량** | 20편 → 3편으로 축약하는 과정에서 "정보의 바다에서 신호를 추출하는" Intelligence Coordinator 역량 |
| **실용적 산출물** | 수업 후에도 매일 실제로 사용할 수 있는 개인화된 논문 브리핑 도구를 보유 |

- **왜 압도적인가?** 이 시나리오의 킬링 포인트는 **"수업이 끝나도 계속 쓴다"**는 것입니다. 다른 시나리오의 산출물은 한 번 만들고 끝이지만, 이 뉴스레터 앱은 학생이 매일 아침 열어보는 **살아있는 도구**가 됩니다. 그리고 매일 TOP 3를 고르는 행위를 반복하면서, 학기가 끝날 때쯤에는 **"뇌과학 논문의 품질을 직감적으로 평가할 수 있는 눈"**이 길러집니다. 가장 낮은 난이도로 시작하지만, 확장 과제까지 가면 **실무급 데이터 파이프라인**을 경험하게 되는 것도 매력 포인트입니다. 또한 이것은 Phase 0의 논문 수집 스킬과 직접 연결되므로, "0단계에서 배운 것을 실생활 도구로 승화"하는 완벽한 연결고리가 됩니다.

### 🎙️ Phase 3. 에이전틱 브로드캐스팅 (NotebookLM 팟캐스트 제작 및 론칭)

위의 시나리오 중 하나를 완수하여 얻은 **최종 결과물(요약된 논문 집합, JSON 데이터베이스, 시뮬레이터 대본, 연구 계획서 등)**을 나만의 구글 `NotebookLM`에 전부 업로드합니다.

- **작업 내용:**
  1. NotebookLM의 **'오디오 개요(Audio Overview)'** 기능을 활용하여, 본인이 구축한 지식 베이스를 바탕으로 한 10분짜리 심층 팟캐스트를 생성합니다.
  2. 단순 생성이 아니라, NotebookLM의 **'지시사항(Instructions)'** 기능을 활용하여 팟캐스트 호스트들의 논의 방향을 통제합니다 (예: "호스트 A는 상징주의를 옹호하고, 호스트 B는 연결주의를 옹호하며 격렬하게 논쟁해 줘", "내 스크린타임 데이터를 보고 호스트들이 경악하면서 팩트폭력을 날려줘").
- **의의:**
  학생들은 단순히 코드를 짜거나 텍스트를 요약하는 데서 그치지 않고, 본인이 구축한 에이전틱 시스템의 결과물이 대중적인 오디오 콘텐츠로 가공·확장되는 지식 생산의 **전 주기(Full Cycle)**를 경험하게 됩니다.

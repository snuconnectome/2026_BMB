# 2주차 과제: 에이전틱 실습 결과 제출

**마감:** 다음 수업 전까지
**제출:** GitHub 저장소 — `week02_agentic_practice/` 폴더

---

## 뭘 제출하나요?

수업 시간에 AI(Cursor 등)와 함께 실습한 **결과물**, **배운 점 정리**, **대화 기록** 세 가지입니다.

---

## 제출물

### 1. 실습 산출물

수업에서 만든 파일들을 `outputs/` 폴더에 그대로 넣으세요.
수업 중 못 끝낸 부분은 이어서 완성해도 됩니다.

### 2. 학습 보고서 (`report.md`)

아래 질문에 **자기 말로** 답하세요. 길게 쓸 필요 없습니다.

```markdown
# 2주차 학습 보고서

## 어떤 시나리오를 선택했나요?
- 시나리오 번호/이름:
- 고른 이유:

## 새로 알게 된 것
- 실습 전에는 몰랐는데 AI와 대화하면서 알게 된 것:
- 특히 흥미로웠거나 의외였던 점:

## 어디까지 탐험했나요?
- 처음에 던진 질문:
- 대화가 깊어지면서 도달한 질문:

## AI 답변에 대한 판단
- AI 답변 중 의심스럽거나 확인이 필요하다고 느낀 부분:
- 팩트체크를 시도했다면 그 결과:

## 아직 궁금한 것
- 실습 후에도 남아 있는 질문:
```

> AI가 써 준 문장을 그대로 붙여넣지 마세요. 본인이 이해한 만큼만, 자기 말로 쓰세요. "아직 잘 모르겠다"고 솔직하게 쓰는 게 복붙보다 좋은 점수를 받습니다.

### 3. 대화 기록 (`audit_log.md`)

AI와 나눈 **전체 대화**를 그대로 저장합니다.
내가 입력한 프롬프트와 AI 응답이 모두 포함되어야 합니다.

## FAQ

**Q. 수업 시간에 못 끝냈어요.**
A. 이어서 완성하면 됩니다. 추가 대화도 audit_log에 포함하세요.

**Q. 대화 기록을 저장 안 했어요.**
A. Cursor 히스토리에서 복구해 보세요. 안 되면 같은 시나리오를 다시 해 보고 그 대화를 제출하세요.

**Q. 만든 앱/웹페이지가 안 돌아가요.**
A. 완성도보다 과정이 중요합니다. 에러를 고치려고 시도한 과정 자체가 학습입니다.

**Q. Cursor 대신 다른 AI를 썼어요.**
A. 어떤 도구든 상관없습니다. 대화 전체를 저장해서 제출하면 됩니다.

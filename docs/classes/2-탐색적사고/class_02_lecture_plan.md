**Session:** Class 02 - Agentic Onboarding & The Memory Palace

## 🎯 학습 목표 (Learning Objectives)

1. **Agentic Onboarding 완료**: 오딧 로그(Audit Log)의 가치와 작성법을 이해하고, 'Steering(조향)'이 곧 핵심 학습(Action)임을 내재화한다.
2. **Intelligence Coordinator로의 전환**: 단순한 AI '사용자'를 넘어, 스스로 필요한 도구와 파이프라인을 설계하고 창조하는 '에이전틱 크리에이터(Agentic Creator)'의 경험을 한다.
3. **Agentic Builder 실습**: 학부생 수준에서 코딩 없이도 NotebookLM(지식 추출)과 Cursor(앱 빌더)를 연동하여 나만의 뇌과학 학습 보조 앱(예: 플래시카드 웹앱)을 30분 만에 기획하고 대화형으로 창조해 낸다.

## ⏱️ Master Chronometry (Total Duration: 75 Minutes)

### Prologue: Administrative & Epistemic Setup (0:00 - 0:05 | 5 mins)

- **0:00 - 0:05 | NotebookLM 권한 셋업:**
  - 수업 시작과 동시에 학생들의 Gmail 계정을 수집합니다 (구글 폼 또는 QR 코드 활용).
  - 수집된 이메일을 바탕으로 2026 BMB 전용 NotebookLM('2026 BMB (Agentic Clone)')에 뷰어/에디터 권한을 실시간으로 부여합니다.

### Movement 1: The Final Boss & Agentic AI Onboarding (0:05 - 0:25 | 20 mins)

*(Class 01 오리엔테이션 보충 진도)*

- **0:05 - 0:13 | Steering as Action:**
  - AI를 교정하는 과정 자체가 가장 가치 있는 학습임을 강조합니다. (AI의 1차 답변을 수용하는 것은 죄악)
- **0:13 - 0:20 | The Final Boss: Audit Log:**
  - 성적의 60%를 차지하는 오딧 로그 평가 기준을 상세히 안내합니다.
  - 훌륭한 오딧 로그의 예시(의도적 예측 오류 창출, 프롬프트 체인을 통한 가설 업데이트)를 보여줍니다.
- **0:20 - 0:25 | 과제 브리핑 (AlphaGo):**
  - 알파고 다큐멘터리 시청 관찰 과제를 리뷰하며, 3단계 AI 스파링 과정(데이터 수집 -> 분석 -> 에세이)을 어떻게 진행해야 하는지 가이드합니다.

### Movement 2: Paradigm Shift - Agentic Creator (0:25 - 0:35 | 10 mins)

- **0:25 - 0:35 | Tool Consumer vs. Agentic Creator:**
  - 챗GPT나 NotebookLM에 이미 있는 정보를 묻는 '소비자(Consumer)' 단계에서 벗어나야 함을 역설합니다.
  - "진정한 에이전시는 내가 원하는 지식 파이프라인이나 프로덕트(App/Newsletter)를 기계에게 '만들게' 지시하는 데에서 나옵니다."
  - 오늘 실습의 의미: 코딩을 몰라도, 과거 90년대 세상을 놀라게 한 연구와 오늘의 최신 연구를 비교 분석해 주는 나만의 '뉴스레터/웹앱'을 30분 만에 뚝딱 만들어 봅니다.

### Movement 3: Live Practice - Cursor x NotebookLM 에이전틱 빌딩 (0:35 - 0:65 | 30 mins)

*(에이전트 AI를 처음 접하는 학생들을 위한 학습 효율 극대화 실습)*

- **0:35 - 0:45 | Step 1. 지식의 추출 (10분) - NotebookLM의 마법:**
  - 미리 준비된 '2026 BMB NotebookLM' 공용 노트북에 접속합니다.
  - 다음 프롬프트를 통해 방대한 논문/교안에서 필요한 능동적 추론(Active Inference) 관련 핵심 지식을 뽑아냅니다.
  - *"강의 자료를 바탕으로 능동적 추론과 예측 오류의 핵심 개념을 학부생 수준으로 쉽게 설명하고, 학습 여부를 테스트할 객관식/주관식 질문 5개를 생성해 줘."*
  - 출력된 요약과 질문 5개를 복사(Copy)하여 로드아웃을 준비합니다. (RAG 시스템을 직접 구축하지 않고 구글에 100% 위임)

- **0:45 - 0:55 | Step 2. 에이전틱 빌딩 (10분) - Cursor로 나만의 학습 앱 만들기:**
  - Cursor IDE를 열고 일제히 비어있는 폴더에 Composer(Cmd+I) 창을 켭니다.
  - NotebookLM에서 추출한 지식을 붙여넣고 Cursor에게 다음과 같이 지시합니다.
  - *"복사한 텍스트(요약+질문)를 활용해서, 내가 오늘 배운 내용을 복습할 수 있는 모던한 다크모드 '인터랙티브 플래시카드 웹앱(index.html)'을 플립(Flip) 애니메이션과 함께 만들어 줘."*
  - 코딩 경험이 없는 학생들이 1분 만에 브라우저에서 자신이 만든 웹앱이 구동되는 경험을 실시간으로 확인합니다.

- **0:55 - 0:65 | Step 3. 스티어링과 질문의 심화 (10분) - 학습의 질 끌어올리기:**
  - 생성된 플래시카드 앱을 써보며 디자인이나 기능을 수정(Steering)하는 과정을 거칩니다.
  - 수동적 논문 읽기 시간을 소프트웨어 생성으로 단축했으므로, 남은 시간에 Cursor와 더욱 깊은 질의응답을 나눕니다. (예: "만약 우울증 환자라면 이러한 예측 오류를 받아들이는 뇌의 방식이 일반인과 다를까?")
  - Cursor가 단순 코드 생성기를 넘어 강력한 도메인 튜터 봇으로 작동함을 체감합니다.

### Epilogue: 지식 궁전(Memory Palace)의 체화 (0:65 - 0:75 | 10 mins)

- **0:65 - 0:75 | Debrief & Q&A:**
  - 단순 문서 요약은 NotebookLM에, 결과물 창조는 Cursor에 위임하는 명확한 **'역할 분리'**의 의미를 토론합니다.
  - 학습 자료를 인터랙티브 소프트웨어 설계 관점으로 재구성해보는 것 자체가 어떻게 강력한 '메타 인지' 학습법이 되는지 강의 전체를 랩업하며 마무리합니다.

## 🛠️ [부록] Cursor + 수업용 DeepSeek 설정 가이드 (학생 배포용)

수업용 API(LiteLLM)를 Cursor에서 쓰려면 아래 순서대로 하면 됩니다.

### 1. Cursor 설치 (이미 있으면 생략)

1. [cursor.com](https://www.cursor.com) 접속 → 본인 OS용 **다운로드**
2. 설치 후 실행 → **이메일 또는 GitHub**로 무료 계정 생성·로그인

### 2. Custom model 선택 후 값 입력

1. **Settings** (⌘+, / Ctrl+,) 열기 → **Models** 또는 **Features** → **API Keys** / **OpenAI** 로 이동
2. 모델 옵션에서 **Custom model** (또는 **OpenAI-compatible** / **Use custom API**) 을 선택
3. 아래 값을 입력합니다.

| 항목 | 입력 |
|------|------|
| **OpenAI API Key** | ON 으로 두고, 조교가 준 **개인 API 키** (`sk-...`) 붙여넣기 |
| **Override OpenAI Base URL** | ON 으로 두고, `https://cadgily-unobstinate-breana.ngrok-free.dev/v1` |
| **Model** | `deepseek/deepseek-v3.2` 또는 `deepseek/deepseek-r1-distill-qwen-32b` |

1. **Save** / **Verify** 후 설정 창 닫기.

### 3. 확인

Cursor 채팅에서 짧은 메시지(예: "안녕") 보내서 응답이 오면 성공입니다.

- **401에러** → API 키 확인 (오타·공백 없이)
- **연결 실패** → Base URL 끝에 `/v1` 있는지, 조교에게 ngrok 켜져 있는지 문의

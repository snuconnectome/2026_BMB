# Class 3: Deep Inquiry & Agent Orchestration

**과목명**: 뇌, 마음, 행동 (2026)
**강의자**: 차지욱 교수 (서울대학교 커넥톰연구실)

## 1. 실험(Testing)과 피드백 연결하기

2강에서 세운 가설은 아직 우리 머릿속의 '예측(Prediction)'에 불과합니다. 뇌가 행동(Action)을 통해 세상 속에서 예측을 테스트하고 교정하듯, 우리도 **Agentic AI를 도구(Enabling Technology)로 삼아 데이터 세계에서 가설을 테스트합니다.**

## 2. Orchestration 실습: 행동(Action)으로서의 코드 실행

### Activity: 데이터 분석의 주체 되기

AI에게 코딩 문법을 묻는 것이 아닙니다. 자신의 가설 검증에 필요한 분석 파이프라인의 **설계도**를 Cline에게 지시합니다.

**프롬프트(Action) 예시:**
> "나는 지금 '서파 수면 시기 알파파가 인지부하에 미치는 영향'이라는 가설을 검증할 테스트 파이프라인이 필요해. MNE-Python 라이브러리를 사용해서, EDF 포맷의 수면 EEG 데이터를 불러오고, 1~40Hz 필터링 후 ICA를 돌리는 파이썬 스크립트를 작성해서 로컬 환경에 `src/run_eeg_test.py`로 저장해."

### Activity: 에러라는 이름의 피드백(Prediction Error) 수용하기

첫 코드가 완벽하게 돌아갈 확률은 0에 수렴합니다. 에러가 발생했다는 것은 우리(혹은 AI)의 예측이 빗나갔다는 가장 확실한 환경의 피드백입니다.

- 에러 로그를 그대로 Cline에게 전달하여 진단(Re-evaluation)을 요청하세요.
- AI와 끊임없이 티키타카(Iteration)하며 문제를 해결하는 이 피드백 루프 자체가 뇌의 학습(Inside-Out) 원리와 동일합니다.

## 3. 비판적 검토(Audit)

결과가 나왔다고 끝이 아닙니다. 코드가 뱉어낸 숫자가 뇌과학적으로 올바른 지식인지 NotebookLM이나 동료의 리뷰를 통해 다시 한 번 검증하세요.

## 4. 과제

수립된 분석 파이프라인(스크립트)과 그 스크립트가 뱉어낸 '피드백(결과)'을 요약하여 4강 발표 준비를 합니다.

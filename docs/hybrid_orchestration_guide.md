# Hybrid Orchestration Guide: Local Mac ↔ DGX-Spark

이 문서는 2026 BMB 코스 프로젝트에서 로컬 환경(Mac)의 편리함과 서버 환경(DGX-Spark)의 강력한 연산 능력을 결합하여 사용하는 방법을 설명합니다.

## 🌉 하이브리드 아키텍처 개요

- **로컬 (Mac)**: 코드 작성, 데이터 시각화, AI 에이전트와의 대화, 최종 결과물(NotebookLM) 확인.
- **원격 (DGX-Spark)**: 무거운 GPU 연산(임베딩, 모델 추론), 대규모 데이터 다운로드, 복잡한 전처리.
- **지능 (MCP Memory)**: 모든 환경의 맥락을 연결하는 장기 기억 보관소. 지식 그래프를 이용해 세션 간 지식 전이 및 개인화된 답변 제공.

우리는 이 두 환경을 연결하기 위해 **Fabric (Python-based SSH Automation)** 라이브러리를 사용합니다.

---

## 🛠 주요 명령어 (Fabric Tasks)

프로젝트 루트 디렉토리에서 아래 명령어를 실행할 수 있습니다.

### 1. 코드 및 데이터 동기화 (`fab sync`)

로컬의 최신 코드와 문서를 DGX 서버로 업로드합니다.

- 내부적으로 `rsync`를 사용하므로 변경된 파일만 효율적으로 전송합니다.
- `.git`, `node_modules` 등 불필요한 폴더는 자동으로 제외됩니다.

### 2. 원격 인프라 테스트 (`fab remote-test`)

DGX 서버에 접속하여 GPU(NVIDIA) 상태와 파이썬 환경을 체크합니다.

### 3. 하이브리드 RAG 빌드 (`fab build-rag`)

논문이나 수업 자료를 DGX 서버의 GPU를 이용해 빠르게 임베딩합니다.

- 작업 완료 후 생성된 Vector DB 인덱스를 자동으로 로컬 맥북으로 수거해 옵니다.

### 4. 장기 기억 동기화 (MCP Memory Tools)

세션이 바뀌어도 잊지 말아야 할 강의 철학, 학생 정보, 코드 스타일 등을 `memory` 도구를 통해 저장합니다.

- `create_entities`: 새로운 강의 주제나 엔티티 등록
- `add_observations`: 기존 엔티티에 새로운 사실(Observation) 추가
- `read_graph`: 전체 지식 구조 파악

## 🤖 AI 에이전트를 위한 지침 (For AI Agent)

AI 에이전트(Antigravity 등)는 이 시스템을 다음과 같이 활용하여 교수님(사용자)의 작업을 도와야 합니다.

1. **지능적 실행(Dispatch)**: 사용자가 "무거운 작업을 해줘"라고 하면, 에이전트는 로컬 터미널에서 `fab` 명령어를 실행하여 DGX 서버에 작업을 할당합니다.
2. **데이터 수거(Collection)**: 서버 작업이 끝나면 에이전트는 자동으로 결과 파일을 가져와 로컬 지식 베이스(NotebookLM 등)에 업데이트합니다.
3. **컨텍스트 지속성(Memory)**: 중요한 결정이나 교수님의 선호도, 학생별 피드백 등은 즉시 MCP Memory에 엔티티로 등록하여 다음 대화에서 활용합니다.
4. **환경 기억**: 에이전트는 항상 `fabfile.py`를 프로젝트의 '지휘 본부'로 인식하고 최신화해야 합니다.

## 📋 설정 방법 (Initial Setup)

1. **Fabric 설치**: `pip install fabric`
2. **SSH 설정**: `~/.ssh/config` 파일에 `dgx-spark` 호스트가 등록되어 있어야 합니다.
3. **파일 구성**: 프로젝트 루트의 `fabfile.py`가 오케스트레이션 로직을 담당합니다.

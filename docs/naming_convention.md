# BMB 2026: 문서 명명 규칙 및 관리 체계 (System Documentation)

본 과정의 모든 강의 자료와 과제 문서는 학습의 연속성과 구조적 이해를 돕기 위해 다음과 같은 **체계적 명명 규칙(Systematic Naming Convention)**을 따릅니다.

## 📁 파일 명명 원칙

모든 파일은 `class_[주차]_[순서]_[내용].md` 형식을 유지합니다.

1. **[주차] (Class Number)**: 해당 강의가 진행되는 주차 (예: 01, 02, 03...)
2. **[순서] (Sequence Number)**: 해당 주차 내에서의 학습 순서 (예: 1, 2, 3...)
3. **[내용] (Content Description)**: 파일의 핵심 주제 또는 성격 (예: orientation, homework, ai_exploration)

## 🛠️ 관리 지침 (Internal Guidelines)

- **상호 참조(Cross-reference)**: 문서 내에서 다른 문서를 언급할 때, 반드시 변경된 체계적 파일명을 사용하여 링크를 업데이트해야 합니다.
- **버전 관리**: 자료 수정 시 파일명은 유지하되, 문서 내부의 'Update' 날짜를 갱신합니다.
- **신규 자료 추가**: 새로운 주차의 자료를 생성할 때 기존의 `XX_X` 패턴을 엄격히 준수하여 정렬 순서가 흐트러지지 않도록 합니다.

> **Note**: 이 명명 체계는 학생들의 **'Action Machine'**이 혼란 없이 다음 단계의 '예측(Prediction)'을 수행할 수 있도록 돕는 인지적 이정표 역할을 합니다.
